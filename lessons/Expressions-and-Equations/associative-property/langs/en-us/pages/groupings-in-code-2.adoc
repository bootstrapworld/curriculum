== Make a Prediction

We're going to work with two functions: 

- `darker`: Combines two images by comparing the pixels from the two images at each position and selecting the darker pixel
- `blend-images`: Combines two images by averaging the pixel color codes from the two images at each position.

@vspace{1ex}

@n Groupings matter for one of them. Which one would you predict? Why? @fitb{}{} +
@blanklines{1}{}

@n Let's try to generalize our understanding. What do you think makes the @proglang function where the order of the groupins mattered different from the other functions we've considered today? @fitb{}{} +
@blanklines{4}{@ifsoln{`blend-images` is different because the output isn't data from the original images. Instead it's the average of data from 2 of the original images... so when we average it with data from the third image the third image's data gets counted as twice as important. All of the functions on the first page combined two images so that we could still see the original images. `darker` made a new image by choosing a pixel to keep from one of the original images, so when we combined it with a third image it still just chose the darkest of the three original pixels.}}

@star Can you identify other @proglang functions or operations for which groupings don't matter? @fitb{}{@ifsoln{`blend-images`, `+`, `*`}}
