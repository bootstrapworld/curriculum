== Programming Exploration: Does Order Matter?

=== Overview
Students explore whether various image-producing @proglang functions are commutative.

=== Launch
@slidebreak{Launch}

@lesson-instruction{
- Turn to @printable-exercise{commutativity-and-code.adoc} and open the @starter-file{expressions-equations}.
- For each function, write a second expression that changes the order of the arguments. Take a minute to predict what image it will return. Finally, test your code in @proglang and sketch the results.
}

@teacher{As students work, encourage them to _always_ make predictions before testing the code. Similarly, the activity will be more valuable if students discuss _why_ the code did or did not produce identical images. Debrief to ensure comprehension.

When everyone is finished, check in with students. Did everyone discover that _none_ of these functions were commutative? There is a good chance your students will wonder if _any_ @proglang functions are commutative!
}

=== Investigate
@slidebreak{Investigate-DN}

In the last activity, we discovered that we couldn't change the order of the arguments for any of the image-producing functions on the page! If order matters for @show{(code 'beside)}, @show{(code 'above)} and @show{(code 'overlay)} does that mean it will matter for all @proglang functions? Let's continue exploring!

@lesson-instruction{
- Turn to @printable-exercise{commutativity-and-code2.adoc}.
- With your partner, test @show{(code 'string-contains)}, @show{(code 'blend-images)}, @show{(code 'lighter)} and @show{(code 'darker)} to see whether the order of the arguments matters.
}

@teacher{
Reminder: Color blending in @proglang does not behave exactly as it would in the real world, because computers typically use "RGB" (red, blue, green) to express color. On a primary color wheel blue and yellow would blend to be green. In RGB...the results might surprise you.
}

== The Commutative Property

=== Overview
Students begin by trying to come up with counterexamples to characters' arguments that each of the operations is commutative. Then we will use the Circles of Evaluation to reveal the structure of expressions containing three numbers and consider how commutativity works for them.

=== Launch
@slidebreak{Launch}

When order doesn't matter to the outcome of a mathematical expression, we call an operation @vocab{commutative}.

@lesson-instruction{
- Does order matter when we add? subtract? multiply? divide? Let's take a moment to think about which operations are commutative. 
- Complete @printable-exercise{discover-comm.adoc}. 
}

@slidebreak{Launch}

@QandA{
@Q{Which operations were commutative?}
@A{Just addition and multiplication.}

@Q{If any of the operations weren't commutative, was there a pattern to what happened when you changed the order of the arguments?}
@A{For subtraction changing the order changes the sign of the answer! (unless both numbers are the same).}
@A{If we're dividing counting numbers (and the numbers aren't the same), changing the order changes the answer from a whole number to a fraction or vice versa!}
}

=== Investigate
@slidebreak{Investigate}

@lesson-instruction{
- Let's dig a little deeper into commutativity. What do you think will happen if there are more than two numbers in an expression?
- Complete @printable-exercise{discover-comm-2.adoc} and @printable-exercise{discover-comm-3.adoc}
}

@slidebreak{Launch}

@QandA{
@Q{Are multiplication and addition commutative when there are more than 2 numbers?}
@A{If the problem only involves addition or only involves multiplication, yes!}
@A{For problems that involve both addition and multiplication, we have to use the Order of Operations to identify the groupings first. Then we can apply the commutative property as long as we don't change the groupings.}
}

@slidebreak{Launch}

@lesson-point{
For any expression involving only addition or only multiplication, changing the order of the numbers will not change the result.
}

@teacher{
Ensure comprehension of this key idea, as students will apply their understanding of the Commutative Property throughout the remainder of the lesson.
}

@lesson-instruction{
- Turn to @printable-exercise{comm-prop-table.adoc}. You will see Circles of Evaluation involving all of the different operations. 
- Paying careful attention to which operations in each problem are commutative, use the Commmutative Property to draw as many different equivalent Circles of Evaluation as you can for each expression. 
}

@teacher{
As students work, draw their attention to the operation as they rearrange the values inside the Circles of Evaluation, helping to reinforce that the Commutative Property does not apply for division and subtraction.
}

@slidebreak{Investigate}

@QandA{
@Q{How come some of the expressions generated fewer equivalent Circles of Evaluation than others?}
@A{Division and subtraction aren't commutative so expression with those operations resulted in fewer (or no) equivalent expressions.}

@Q{Were you able to find equivalent expressions for all of the Circles of Evaluation?}
@A{Not the last one. It just had subtraction and division, so it didn't include any commutative operations.}

@Q{What would have to be true of a four-number expression for it to have more than four equivalent Circles of Evaluation?}
@A{It would only use addition and/or multiplication so that each group could swap orders and the order of the two groups could also be swapped, giving us 8 equivalent expressions.}
}

=== Common Misconception
Students may attempt to apply other properties when developing equivalent Circles. In such instances, help students to see that their Circle of Evaluation is indeed equivalent, but is not an example of the Commutative Property at work.

=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{At the beginning of the lesson we explored the functions `above`, `beside`, `blend-images`, `darker`, `lighter`, `overlay`, and  `string-contains`. Which could we describe as commmutative?}
@A{`blend-images`, `darker` and `lighter`}

@Q{Which operations are commutative?}
@A{addition and multiplication}

@Q{How can we apply the commutative property to expressions that include both operations that are commutative and others that are not commutative?}
@A{Regardless of how  many things are happening in an expression, as long as we identify the groupings before we begin and don't change them, we can change the order in which terms that are being added are added and we can change the order in which terms that are being multiplied are multiplied.}

@Q{Summarize the Commutative Property in your own words.}

@Q{How might a strong understanding of the Commutative Property be useful when you're doing computations in your head?}
}
