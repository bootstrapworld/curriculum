==== The Red Green Blue Color Model
@slidebreak{Investigate}

You have probably learned about the primary color wheel in art class, where the color across from blue is orange. Computers use a different color logic. As a result, the @vocab{inverse} of a color in @proglang is not the color that is located across the primary color wheel, but the color that is across from it on the Red Green Blue Color Model.

@indented{
[cols="1,^.^10a,2,^.^9a,2", grid="none", frame="none", options="header"]
|===
|| Red Yellow Blue Color Star 	        || Red Green Blue Color Model  |
|| @image{images/color-star.png, 80%}   || @image{images/rgb-venn-diagram.png, 80%} |
|===
}

@lesson-point{@proglang generates colors by adding red, green and blue pixels in specified ratios.} 

Those ratios are represented in RGB codes as integer values ranging from 0 to 255. The @vocab{inverse} of any color can be found by finding the ratio that complements the original color's RGB ratio such that all 255 parts of red, green and blue are accounted for. For example: 

@indented{
[.data-table, cols="3a,1a,11a", options="header"]
|===
|Color                         |Code            |
|Red                           |RGB(255,0,0)    |uses all of the possible red and no blue or green
|Inverse of red: +
cyan/aqua                      |RGB(0,255,255)  |doesn't use any red, and uses all of the possible blue and green. It is the color across from red on the RGB model
|Royal blue                    |RGB(65,105,225) |uses 25.5% of the possible red, 41.2% of the possible green, and 88.2% of the possible blue
|Inverse of royal blue: +  
tannish                        |RGB(190,150,30) |uses the other 74.5% of possible red, the other 58.8% of the possible green and the other 11.8% of the possible blue
|===
}

@A{Each pixel has the opposite RGB value:

== The Additive Inverse Property

=== Overview
Students discover the Additive Inverse Property, which tells us that adding a number and its opposite always produces zero.

=== Launch
@slidebreak{Launch}

What if there was a way to _rewrite_ subtraction as addition? 

@slidebreak{Launch}

I'm going to draw a two-column table on the board. I'm labeling the left-hand side "Starting Value," and I'm labeling the right-hand side "Additive Inverse".

I'm going to write a number in the left-hand column. You are going to tell me what value I should add to that number, to get a sum of zero. I'll record your response in the right-hand column.

[cols="^1,^1", options="header"]
|===
| Starting Value	  |  Additive Inverse
| 5           		  |
| 20                |
|@math{\frac{1}{2}} |
|@math{-45}         |
|===

@slidebreak{Launch}

@QandA{
@Q{The first number is @math{5}. What do I need to add to @math{5} to get @math{0}?}
@A{@math{-5}.} 

@Q{The next number is @math{20}. What do I need to add to @math{20} to get @math{0}? ...}

@Q{Can someone offer me another pair of numbers that add up to @math{0}?}
}

@teacher{Allow a variety of students to share. Record responses on the table.}

@slidebreak{Launch}

These number pairs all represent @vocab{opposites}.

@lesson-point{
The @vocab{Additive Inverse Property} tells us that adding a number and its opposite always produces zero.
}

Every number has an additive inverse.

@teacher{
Ensure that students are comfortable with this concept before moving onto the next activity.
}

== Addition and Subtraction: Inverse Operations

=== Overview
Students rewrite addition expressions as subtraction, and subtraction expressions as addition by applying their knowledge of the @vocab{Additive Inverse Property}.

=== Launch
@slidebreak{Launch}

At some point we learned that we should always add and subtract from left to right. 

Let's consider the expression @big{@math{16 
+ 973 - 972}}. What would happen if we didn't work from left to right?

@indented{
[cols="^1a,^1a"]
|===
|@show{(coe '(- (+ 16 973) 972))} = @show{(coe '(- 989 972))} = @show{(coe '(17))}
|@show{(coe '(+ 16 (- 973 972)))} = @show{(coe '(+ 16 1))} = @show{(coe '(17))}
|===
}

@QandA{
@Q{Is it essential to evaluate this expression from left to right?}
@A{No. They both returned the same answer!}

@Q{If you knew that you'd get the correct answer by subtracting first, why might you rather compute in that order?}
@A{The second order makes it much easier to do the computations in our head!}
}

@slidebreak{Launch}

The tricky thing is that order _sometimes_ matters. 

- We can evaluate as if order _always_ matters. We will _always get the correct answer_ this way, but we might end up working harder.
- We can try to learn a more nuanced pattern...

@lesson-instruction{
- Complete @printable-exercise{is-there-another-way.adoc}.
- Be prepared to share what you Notice and Wonder.
}

@slidebreak{Launch}

@QandA{
@Q{What pattern did you find?}
@A{In an expression with 3 numbers that involves addition and subtraction
    * if it starts with addition, the order we compute in doesn't matter
    * if it starts with subtraction, we have to compute from left to right
}

@Q{Would it be easy to apply this pattern to an expression with 4 numbers? 5 numbers? 8 numbers?}
@A{No. The pattern we just described is for expressions with 3 Numbers and we'd have to build a more complex set of rules if we wanted to apply it to longer expressions.}

@Q{Why do you think students often learn that they have to add and subtract from left to right, even though another order is _sometimes_ possible?}
@A{It's easier to keep track of a single rule that always work than to remember a long list of rules.}
}

@slidebreak{Launch}

@lesson-point{
Subtraction is less flexible than addition.
}

=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{Claire and Soraya want to write an equivalent expression for @math{22 - 30}. Soraya grabs a pencil and writes the following: @math{22 + -30}. Claire writes: @math{30 - 22}. Are they both correct? Explain.}
@A{Soraya correctly applied the additive inverse.}
@A{Claire's expression would evaluate to the opposite of the original expression, because the pull toward the negative side has shifted from 30 to 22.}

@Q{How could we use the Additive Inverse Property to simplify this expression using mental computation: @math{3 + 96.8 - 42.74 - 96.8 + 7 - 3 + 42.74}}
