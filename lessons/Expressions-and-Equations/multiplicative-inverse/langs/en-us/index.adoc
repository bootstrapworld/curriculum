== Programming Exploration: Scaling Up and Back

=== Overview
Students explore the `scale` function, identifying what happens when we scale an image and what we would have to do scaled image in order to get back to the original image.

@slidebreak{Launch}

@QandA{
@Q{What did  @show{(code '(scale 0.5 (scale 2 pyret)))}   return? Why?}
@A{The original image.}
@A{Because doubling and then halving canceled each other out.}

@Q{What scale factor do you think we would apply to get an image the size of an original after it's been scaled down using   @show{(code '(scale 1/5 ...))}? Explain your thinking.}
@A{@show{(code '(scale 5 ...))}}
@A{Possible response might include:
	* the new image will be little so we need to make it bigger again
	* @math{1/5} is the same as dividing by 5, so we need to multiply by 5 to undo it
	* 5 is the reciprocal of 1/5
	* @math{1/5 \times 5 = 1} (No need to give this one away if it doesn't surface!)
  }
}

== The Multiplicative Inverse Property

=== Overview
Students discover the Multiplicative Inverse Property, which tells us that multiplying a number and its reciprocal always produces one.

=== Launch
@slidebreak{Launch}

The thinking we just did about scaling images up and back involved "undoing" multiplication with @vocab{multiplicative inverses}.

@lesson-point{Any number multiplied by its multiplicative inverse will equal 1 _(the multiplicative identity)_.}

@slidebreak{Launch}

@teacher{Draw or project the following table on the board.}

[cols="^1,^1", options="header"]
|===
| Starting Value					| Reciprocal
| @math{\frac{1}{2}}			|
| @math{\frac{1}{3}}			|
| @math{\frac{1}{100}}		|
|	@math{10}								|
| @vspace{3ex}						|
|===

@lesson-instruction{
For each number in the left-hand column of this table, your job is to tell me the @vocab{multiplicative inverse}: the value I should _multiply_ my starting number by to get a product of 1. I'll record your responses in the right-hand column.
}

@QandA{
@Q{The first number is @math{\frac{1}{2}}. What can I multiply that by to get 1?}
@A{2}

@Q{The second number is @math{\frac{1}{3}}. What can I multiply that be to get 1?...}

@Q{What's another pair of numbers -- a fraction and a whole number -- that multiply together to produce 1?}
@A{Answers will vary}

@Q{@opt{What could we multiply @math{\frac{2}{3}} by to get 1?}}
}

@teacher{Allow a variety of students to share. Record responses on the table.}

@slidebreak{Launch}

The number pairs we just looked at are all multiplicative inverses, which can also be called @vocab{reciprocals}. 

- The reciprocal of any integer @math{n} is @math{\frac{1}{n}}. 
- The reciprocal of any fraction @math{\frac{a}{b}} is @math{\frac{b}{a}}.

@lesson-point{
The @vocab{Multiplicative Inverse Property} tells us that multiplying a number by its @vocab{reciprocal} always produces one.
}

Every number -- except zero -- has a reciprocal.

@strategy{Why doesn't zero have a reciprocal?}{
No number can by multiplied by zero to produce 1. (Zero times anything is always zero!) +
Since there is no reciprocal for zero, division by zero is undefined.
}

=== Investigate
@slidebreak{Investigate}

@lesson-instruction{
- Turn to @printable-exercise{multiplicative-inverse.adoc}.
- In the first section, practice finding reciprocals and write them in the space provided.
- Then, fill in the missing number to complete the equations. Some equations use mathematical notation and some use Circles of Evaluation.
}

=== Synthesize
@slidebreak{Synthesize}

- How would you explain the Multiplicative Inverse Property to another student?
- Can you think of a way to visually represent that a number multiplied by its reciprocal produces one?

== Multiplication and Division: Inverse Operations

=== Overview

Students rewrite multiplication expressions as division, and division expressions as multiplication by applying their knowledge of the @vocab{Multiplicative Inverse Property}.

=== Launch
@slidebreak{Launch}

At some point we learned that we should always multiply and divide from left to right. 

Let's consider the expression @big{@math{100 \times 20 \div 5}}. What would happen if we didn't work from left to right?

|===
|@show{(coe '(/ (* 100 20) 5))} = @show{(coe '(/ 2000 5))} = @show{(coe '(400))}
|@show{(coe '(* 100 (/ 20 5)))} = @show{(coe '(* 100 4))} = @show{(coe '(400))}
|===
}

@QandA{
@Q{Is it essential to evaluate this expression from left to right?}
@A{No. They both returned the same answer!}

@Q{If you knew that you'd get the correct answer by dividing first, why might you rather compute in that order?}
@A{The second order involves smaller numbers, making it easier to do the computations in our head!}
}

@slidebreak{Launch}

The tricky thing is that order _sometimes_ matters. 

- We can evaluate as if order _always_ matters. We will _always get the correct answer_ this way, but we might end up working harder.
- We can try to learn a more nuanced pattern...

@lesson-instruction{
- Complete @printable-exercise{is-there-another-way.adoc}.
- Be prepared to share what you Notice and Wonder.
}

@slidebreak{Launch}

@QandA{
@Q{What pattern did you find?}
@A{In an expression with 3 numbers that involves multiplication and division
    * if it starts with multiplication, the order we compute in doesn't matter
    * if it starts with division, we have to compute from left to right
}
}

@slidebreak{Launch}

@right{@image{images/kenya-flag.png, 150}} In Kenya, students are taught that in expressions with two operators, the first being multiplication and the second being division, they must divide first... and then multiply! 

@QandA{
@Q{What is an advantage to learning to recognize @big{@math{1234 \times 880 \div 88}} as an expression where we can divide first?}
@A{Most people can't multiply @math{1234 \times 880} in their heads.}
@A{If we start with @math{880 \div 88 = 10}, it's easy to multiply @math{1234 \times 10 = 12340}}
}

@teacher{
Learn more about the "Kenya algorithm" in @citation{bay-williams-martinie-2015, "this article"} by Bay-Williams and Martinie (2015). The authors explore the idea that when we teach Order of Operations in a rigid way, "students miss out on opportunities to look for efficient approaches, a critical component of procedural fluency."
}

@slidebreak{Launch}

@QandA{
Let's put our new knowledge to use! Scan each expression to determine the simplest solving strategy, then compute mentally.

@Q{ @math{114 \times 17 \div 17}}
@A{@math{114}}

@Q{@math{15 \times 45 \div 15}}
@A{@math{45}}

@Q{@math{2 \times 16 \times \frac{1}{27} \times 27}}
@A{@math{32}}
}

@slidebreak{Launch}

@QandA{
@Q{Could we apply the "Kenya algorithm" to an expression with 4 numbers? 5 numbers? 8 numbers?}
@A{No. The pattern we just described is for expressions with 3 Numbers and we'd have to build a more complex set of rules if we wanted to apply it to longer expressions.}

@Q{Why do you think students often learn that they have to multiply and divide from left to right, even though another order is _sometimes_ possible?}
@A{It's easier to keep track of a single rule that always work than to remember a long list of rules.}
}

=== Investigate
@slidebreak{Investigate}

We've seen more than once that the order in which we multiply is more flexible than the order in which we divide. What if we could write divisiion problems as multiplication?

@lesson-instruction{
- Scooter claims that we can rewrite division by an integer as multiplying by a unit fraction. To support her claim she asserts that _twelve divided by four_ is the same as _one fourth of twelve_.
- Complete @printable-exercise{discover-inverse.adoc} to explore her claim.
- What did you observe about the multiplicative inverse?
}

@slidebreak{Investigate}

During this exploration, you should have discovered that:

@lesson-point{Any expression involving division can be rewritten as multiplication by the @vocab{reciprocal}.}

- The reciprocal of any unit fraction (@math{\frac{1}{a}}) is an integer (it's denominator (@math{a}).
- Multiplying by a unit fraction (@math{\frac{1}{a}}) produces the same result as _dividing by its denominator_ (@math{a}).
- Dividing by a unit fraction (@math{\frac{1}{a}}) produces the same result as _multiplying by its denominator_ (@math{a}).

@lesson-point{Expressions involving multiplication or division by a unit fraction (a fraction with a numerator of @math{1}) can be rewritten without fractions!}

@slidebreak{Investigate}

@QandA{
@Q{How could using the Multiplicative Inverse to rewrite expressions increase the flexibility of the order in which we do computations?}
@A{We know that we can mulitply in any order! So if we can rewrite division as multiplication, we can compute in any order, without having to remember lots of special rules!}

@Q{Knowing what you know now, what would be the simplest way to compute the value of @math{30 \times 41 \div 6} in your head?}
@A{If we think about it all as multiplication, we can 
  * reorder it as @math{30 \times \frac{1}{6} \times 41}
  * divide @math{30 \div 6} to get @math{5}
  * then multiply @math{5 \times 41} to get @math{205}
}
}

@slidebreak{Investigate}

@teacher{Now, students are ready to continue their exploration of multiplication as the inverse of division.}

@lesson-instruction{
Complete @printable-exercise{which-not-equivalent.adoc}.
}

@slidebreak{Investigate}

@QandA{
@Q{Claire and Soraya want to write an equivalent expression for @math{45 \div 9}. Soraya grabs a pencil and writes the following: @math{45 \times \frac{1}{9}}. Does that make sense?}
@A{Sample response: Instead of dividing by 9, Soraya is multiplying by the reciprocal.}
@A{Yes, Soraya has written an equivalent expression.}
@A{What's the point? The computation will not be any simpler.}
}

@teacher{Invite students to share which problems were most challenging, and which ones felt simple. Have students share strategies for determining equivalence.}
