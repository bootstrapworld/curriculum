== Programming Exploration: Identity 

=== Overview
Students explore @proglang functions that take in an image and produce an image identical to the original.

=== Launch

@QandA{
@Q{@right{@image{images/dog.png, 125}}What functions do you know that can be used to transform an image?}
@A{Sample responses: `scale`, `rotate`, `overlay`, etc...}
}

@teacher{
Students will be working in @proglang soon -- but we recommend keeping things unplugged for now!
}

@QandA{
@Q{Can you think of a way that we can transform this image of a dog that will result in the *exact same image*?}
@A{Solicit student responses and record them on the board. If students do not volunteer answers, consider sharing one or two of the following possible response (but don't give them all away!): 
  * flip it vertically twice
  * flip it horizontally twice
  * rotate it 360 degrees clockwise
  * rotate it 360 counter-clockwise
  * scale it by 1
  * slide it some distance and then return it to its original position
  }
}

@slidebreak{Investigate-DN}

@lesson-instruction{
- Open the @starter-file{expressions-equations} and click “Run.”
- Type `dog` into the Interactions Area.
- Take a few minutes to explore. How many “do-nothing” transformations can you find for generating identical images?
- Up for a challenge? Try applying multiple “do-nothing” functions to the `dog` image.
}

@slidebreak{Investigate-DN}

Now that you've had a chance to experiment, let's explore whether a few specific @proglang functions can be applied to "do-nothing".

@lesson-instruction{
- Turn to @printable-exercise{scale-rotate-identity.adoc}.
- As you work through this activity, be sure to make predictions about the code before testing it out!
}

@slidebreak{Investigate}

@QandA{
What did you discover? 
@Q{For what values did `scale` produce identical images of the dog?}
@A{Scaling by 1.}

@Q{For what values did `rotate` produce identical images of the dog?}
@A{Rotating by 360, -360, or any other multiple of 360}

@Q{For what values did `rotate` produce identical images of the dog?}
@A{Any transparent image that wasn't larger than the original image.}

@Q{What can we `overlay` to produce identical images?}
@A{Any transparent image that isn't bigger than dog.}

@Q{Were you able to produce identical images using `flip-vertical` or `flip-horizontal`?}
@A{If we use `flip-vertical` it ends up upside down, but if we `flip-vertical` a second time it puts it back where it started.}
@A{Applying `flip-horizontal` twice also puts the image back where it started.}
@A{We can apply either of the transformations an even number of times because
  * each pair of `flip-vertical` undo each other
  * each pair of `flip-horizontal` undo each other
}
}

=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{Why might it be useful to know which transformations "do nothing"?}
@A{Answers will vary.}
@A{If we know that scaling by 1 doesn't change the image's size, we also know that scaling by more than 1 will make it bigger and less than 1 will make it smaller.}
@A{If we know that flipping a shape twice will get us back to where we started, we 
  * probably won't bother to flip it twice
  * can simplify the code if we see it flipping an image twice
}

@Q{Why might it be useful to be able to expand an expression without changing it?}
@A{There are lots of reasons that an expanded form can be easier to work with!
  * It might make it easier for other people to see our thinking.
  * It might allow us to connect two expressions by revealing what they have in common.
}
}

== The Identity Property

=== Overview

Students determine if pairs of Circles of Evaluation represent equivalent expressions. The Circles of Evaluation make visible the structural changes that occur when we apply the @vocab{Identity Property}.

=== Launch
@slidebreak{Launch}

@teacher{
Build on prior knowledge by reminding students of the category names they created in @lesson-link{equivalence}, and draw their attention to any category names that connect to the Identity Property. For example, a category like “multiplying by one” or "adding zero" is a strong hint that students have an intuition for the @vocab{Identity Property}!)
}

@lesson-instruction{
Complete @printable-exercise{discover-identity.adoc}.
}

@slidebreak{Launch}

@QandA{
@Q{What did you observe about the Identity Property for division? Subtraction?}
@A{Possible responses: I can multiply or divide by @math{1} and get an equivalent result. I can add or subtract @math{0} and get an equivalent result.}
}

@lesson-point{
It will not change the value of an expression to multiply by 1, divide by 1, add 0 or subtract 0.
}

@teacher{Confirm that students understand the main idea about the Identity Property! They will apply this knowledge throughout the lesson.}

=== Investigate
@slidebreak{InvestigateR}

Sometimes, we will encounter the Identity Property in sneakier forms!

@indented{@show{(coe '(+ 27 (- 8 8)))}}

@QandA{
@Q{How does the Circle of Evaluation above illustrate the Identity Property?}
@A{The inner circle, (@math{8 - 8}) is equal to zero. So the final value will just be @math{27}!}
}

@slidebreak{InvestigateC}

@indented{@big{@math{\frac{3}{4} \times \frac{5}{5} = \frac{15}{20}}}}

@QandA{
@Q{How does this equation illustrate the Identity Property?}
@A{@math{\frac{5}{5} = 1} (As does any fraction whose numerator and denominator are the same)}
@A{We can find an infinite number of equivalent fractions by multiplying by 1 in the form a fraction.}
}

@slidebreak{Investigate}

@lesson-point{When we find equivalent fractions, we are applying the Identity property.}

@slidebreak{Investigate}

@lesson-instruction{
- Complete @printable-exercise{identity-prop-table.adoc}, filling in the blanks so that all of the Circles of Evaluation are equivalent.
- Then, complete @printable-exercise{which-not-equivalent.adoc}. Be sure to explain why each Circle of Evaluation you select does not belong with the others!
}

@teacher{
As students explain why various Circles of Evaluation do not belong with the others, encourage them to practice using vocabulary that they have encountered already.
}

=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{What new understanding do you have about the Identity Property?} 
@A{Sample Answers about operations:
  * We can multiply by 1 in any order
  * With division we have to make sure we're dividing by one and not dividing one by a quantity.
  * We can add zero in any order
  * When subtracting we have to make sure we're subtracting zero and not subtracting from zero
  }
@A{It's a good idea to look for expressions that evaluate to zero or one, because they might simplify our work.}
}


== The Identity Property and Variables

=== Overview
Students use their knowledge of the Identity Property to determine whether or not expressions with variables are equivalent.

=== Launch
@slidebreak{Launch}

The Identity Property _also applies to expressions with @vocab{variables}_. Consider these expressions.

[.embedded, cols="^.^3,^.^1,^.^3, ^.^1,^.^3", grid="none", stripes="none" frame="none"]
|===
| @show{(coe  '(+ 3 q))}	| @math{=} | @show{(coe  '(* (+ 3 q) 1))} | @math{=} | @show{(coe  '(+ (+ 3 q) 0))}
|===

These expressions are equivalent for every value we substitute in for @math{q}... but there's no way to test every possible value! The Identity Property guarantees that any expression will be equivalent to itself multiplied by 1 and also to itself with zero added, no matter whether or not we do the computation.

=== Investigate
@slidebreak{Investigate}

@teacher{
Students will now make the transition from numeric values to variables.

If students would like, they may choose values to represent the variables. Early finishers can substitute in numbers of their choosing to confirm that their analyses of the Circles of Evaluation are correct.
}

@lesson-instruction{
- Complete @printable-exercise{true-or-false-variables.adoc} using your knowledge of the Identity Property to determine if the equation represented by the Circles of Evaluation is true or false.
- Decide @printable-exercise{which-not-equivalent-variables.adoc}. Be sure to explain your thinking.
}
