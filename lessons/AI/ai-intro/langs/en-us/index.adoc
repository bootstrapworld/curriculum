== What is AI?
@objective{ai-effect}
@objective{ai-vs-ml}

=== Overview
Students explore different examples of how people use AI every day, including a mix of recommendation systems and classifiers. Students discover the challenges of defining AI, and think about whether AI is just an algorithm, or something more.

@lesson-point{Is it _thinking_, or is it just _following an algorithm_?}
--
@right{@image{images/flow-chart.gif, 400}}
The first spell-checker was written in 1961. It had a 10,000 word dictionary, and it would check every word you typed to see if it was in the dictionary. It wasn't really _thinking_. It was just following an following an @vocab{algorithm} -- a series of steps that tell a computer how to perform a task -- in this case mechanically comparing strings of letters against a long list. And all it could do was tell you if a word _was_ or _wasn't_ spelled correctly.
--

@slidebreak{Launch}

In 1971, however, a new spell-checker arrived from the Stanford Artificial Intelligence Laboratory called SPELL. SPELL wouldn't just tell you if a word was misspelled - it would _suggest alternative words_ that you might have intended! People were *floored*, and swore that AI had arrived. After all, "how did the computer know what we intended to write?"

Was SPELL _thinking_, or was it, too, just _following an algorithm_?

@lesson-instruction{
- Form groups of 2-3.
- How do you think SPELL knew what words to suggest?
- Be ready to share your ideas with the class!
}

@teacher{Give time for productive discussion about possible algorithms.}

@slidebreak{Launch}

Here's SPELL's algorithm for checking a word:

@indented{
(1) If the word is in the dictionary, we're done!

(2) If it's _not_ in the dictionary, find words that are "near" the misspelled word (one letter off, two letters swapped by accident, etc.)

(3) See if any of _those_ words are in the dictionary, and suggest them as alternatives
}
SPELL had no idea what those alternative words meant. It didn't even know what a word _was!_ And it definitely didn't know what people were thinking when they typed. *It was just an algorithm.*

@slidebreak{Launch}

--
@right{@image{images/mccarthy.png, 200}}
After a while, most people stopped calling spell checkers AI. But in the early 2000s, cell phones introduced a feature called "predictive text": when you start typing a sentence, the phone makes suggestions about what the next word should be. People were *floored*, and swore that AI had arrived...but eventually they stopped calling predictive text AI, too.

Computer scientist @link{https://en.wikipedia.org/wiki/John_McCarthy_(computer_scientist), John McCarthy} coined the phrase "AI effect" to describe this phenomenon: humans keep _moving the goalposts_ for what counts as "Artificial Intelligence" and what counts as "Human Intelligence"! In his words, "As soon as it works, no one calls it AI anymore."
--

== Different Kinds of AI

=== Overview
Students learn that AI is actually a collection of technologies involving Machine Learning, and begin thinking about how they might work.

=== Launch
@slidebreak{Launch}

Modern AI is largely driven by a set of techniques called @vocab{Machine Learning} (ML for short). AI relies so deeply on Machine Learning that many computer scientists just call it "AI/ML".

There are many kinds of Machine Learning algorithms out there, and nearly every example we've come up with falls into one of three categories:

[cols="^1a,^1a,^1a"]
|===
| @image{images/prediction-bots.webp, 250}
| @image{images/recommendation-bot.png, 200}
| @image{images/chat-bot.png, 250}

| *Prediction* +
"Given an input, predict an output" +
Facial recognition, Credit Scores, etc. +

| *Similarity* +
"Given an input, find likely matches" +
Spotify, dating apps, etc.

| *Generation* +
"Make up a story - with pictures! - about an elephant" +
ChatGPT, Stable Diffusion, Gemini, etc.
|===

@slidebreak{Launch}

In Machine Learning, the computer analyzes lots of @vocab{data}, and then identifies patterns in that data using statistics. Machine Learning can be applied to an incredibly diverse set of data (text, maps, pictures, movies, songs, etc), to make predictions, find similarity, and even generate new images, stories, etc.

While AI is often spoken about as if it's something "magical", Machine Learning is actually very concrete: it is just a collection of @vocab{algorithms}!

Let's play with some real tools that use AI, and see if we can make sense of how they might be working...

The first line on the left-hand side of the screen tells Pyret to use a provided file, which contains tools we’ll want to use for this course.

`use context shared-gdrive("Bootstrap-...")`

After that, we see a line of code that _defines_ `shelter-sheet` to be a spreadsheet.

`shelter-sheet = load-spreadsheet("https://docs.google.com....")`

This table is loaded from Google Drive, so now Pyret can see the same spreadsheet you do. If you look carefully, you'll notice that the address listed here is the same address as the spreadsheet we just looked at!

@pd-slide{
*What Happens when Pyret loads a spreadsheet?*

- When Pyret imports a sheet, it reads all the cells and builds something it calls a Table.
- When making a table, Pyret checks all of the cells automatically, and holds us accountable for using clean and complete data.
- Each time we sort, filter or change a Table, Pyret makes a new Table. This is beneficial for a variety of reasons!
}

@strategy{What Happens when Pyret loads a spreadsheet?}{
- When Pyret imports a sheet, it reads all the cells and builds something it calls a _Table_. The original spreadsheet is still out there, exactly as it used to be!
- When making a table, Pyret _checks all of the cells automatically_, and holds us accountable for using clean and complete data so that all visualizations can be trusted. Unlike Pyret, spreadsheets allow us to make visualizations out of any collection of cells -- even if those cells are blank, contain mismatched data, or contain data from multiple, unrelated tables -- which can jeopardize our analysis!
- Each time we sort, filter or change a Table, Pyret makes a _new Table_. This means that we can explore one transformation or another, or even multiple transformations together...and can always go back as many steps as we want.  This is a big improvement over working with spreadsheets, where our options are to choose between making "backup" copies of sheets at every step (which means any change to one of those sheets needs to be replicated _by hand_ in all of the other sheets), or modifying the original sheet (which makes it really hard to undo mistakes).
- Checking data by hand and the inability to undo mistakes are problematic for students, teachers, and Data Scientists alike!  While the strictness of programming may outweigh the benefits for younger students, for students who are comfortable writing code, programming is a vastly more powerful and learner-friendly tool for Data Science!
}

@slidebreak{Investigate}

After that, we see the following code:

```
# load the 'pets' sheet as a table called animals-table
animals-table = load-table: name, species, sex, age, fixed, legs, pounds, weeks
  source: pets-sheet.sheet-by-name("pets", true)
end
```

The first line (starting with `#`) is called a @vocab{Comment}. Comments are notes for humans, which the computer ignores. The next line defines a new table called animals-table, (loaded from the shelter-sheet!) and defines column names: `name`, `species`, `sex`, `age`, `fixed`, `legs`, `pounds` and `weeks`. We could use any names we want for these columns, but it’s always a good idea to pick names that make sense!

@lesson-point{
Even if your spreadsheet already has column headers, Pyret requires that you name them in the program itself.
}

|===

| name 		| species | sex 	 | age 	| fixed | legs 	| pounds| weeks

Pyret lets us use many different kinds of data. In the animals table, for example, there are Numbers (the number of legs each animal has), Strings (the species of the animal), and Booleans (whether it is true or false that an animal is fixed).
