== Multiple Regression

=== Overview
Students extend their understanding of single-variable linear regression to multiple regression with two predictors. Using `curve` and `skew`, they see that each scatter plot from the previous lesson was a 2D projection of higher-dimensional data, and that a plane of best fit is the natural extension of a line of best fit.

=== Launch
@slidebreak{LaunchR}

@QandA{
@right{@image{images/projection.webp, 350}}
Take a look at this picture of a cylinder floating in a box.

@Q{What is making the square shadow on the blue wall?}
@A{A light shined at the side of the cylinder casts a square shadow.}

@Q{What is making the circle shadow on the yellow wall?}
@A{A *different* light shined at one end of the cylinder casts a circular shadow.}

@Q{What shape would the shadow be on the floor?}
@A{A square.}

@Q{How can one shape cast such different shadows?}
@A{A 3D object has multiple sides, so different lights cast different shadows.}

@Q{If we only saw the circle shadow, would we know for sure that it was cast by a cylinder like this?}
@A{No. It could be cast by a short or tall cylinder... or even a sphere, a cone, an hour glass, etc.}

@Q{If the _real_ shape is a cylinder, are the circle and square shadows "lying" to us?}
@A{No — they're each just telling a *part* of the story!}
}

@slidebreak{LaunchR}
@right{@image{images/steering-v-curve-lr.png, 300}}

In @lesson-link{regression-models-simple}, we made four scatter plots — one for each candidate explanatory variable. Each showed a 2D relationship between one variable and `steering-angle`.

But `curve`, `skew`, `offset`, and `speed` are all being measured _at the same time_, for the same data point. Each scatter plot we made was showing just one "shadow" of the data — a 2D projection of information that actually exists in more dimensions.
@QandA{
@Q{If `curve` and `skew` are both inputs, and `steering-angle` is the output, how many dimensions would we need to represent all three at once?}
@A{Three! One axis for `curve`, one for `skew`, one for `steering-angle`.}
}

@teacher{Give students a chance to discuss and brainstorm before moving on.}

If we want to consider all three, then we need to consider each snapshot in the training corpus as a 3d point, with values for `curve`, `skew`, _and_ `steering-angle`.

To make _predictions_ in three dimensions, we need to do _regression_ in three dimensions. 

@lesson-point{
Instead of a line of best fit, we're looking for a *plane of best fit*.
}

@scrub{
@slidebreak{LaunchR}

@ifslide{@right{@image{images/plane-of-best-fit.png, 300}}}

- If `curve` and `skew` are both values in a given snapshot, they each explain a _portion_ of the steering angle at that moment
- We need a function that takes in both slopes (@math{m_{curve}} and @math{m_{skew}}) - considering the "amount of pull" each variable has on the steering angle
- @math{b} is the intercept: the predicted steering angle when both inputs are zero
We want Pyret to learn a function with the best possible values for @math{m_{curve}}, @math{m_{skew}}, and @math{b}:

[.big-codeblock]
@big{
`fun curve-skew-predictor(r): (` @math{m_{curve}}` * r["curve"]) + ( `@math{m_{skew}} `* r["skew"]) + b end`
}
}

@teacher{Draw connections to the cylinder-and-shadows image: each 2D scatter plot from the last lesson is a literal "shadow" of the 3D data, cast by collapsing one dimension flat.}

=== Synthesize
@slidebreak{SynthesizeR}
@ifslide{@right{@image{images/plane-of-best-fit.png, 300}}}

@QandA{
@Q{Why does our multiple-regression function have *two* slopes?}
@A{Because we're defining a *plane*. A flat sheet can tip along two different axes independently, so it needs two slopes — one for each.}

@Q{Each scatter plot from the last lesson was a "shadow" of the 3D data. When we looked at `curve` vs. `steering-angle`, in which direction was the light shining?}
@A{Along the `skew` axis — all the variation in `skew` got collapsed into a flat 2D view.}

@Q{Does that mean the `curve` vs. `steering-angle` scatter plot was "wrong"?}
@A{No — it was accurate, just incomplete. It told part of the story.}
}

@slidebreak{Synthesize}

Different fields use different vocabulary for the same ideas:

[cols="^1a,^1a", options="header", stripes="none"]
|===
| In Math... | In Machine Learning...
| Variable   | Parameter
| Slope      | Weight
|===

In math, the "amount of pull" a variable has on the output is its _slope_. In machine learning, the same idea is called a _weight_. When tech companies talk about "releasing model weights" — or refuse to — they mean the learned coefficients of their regression models.

@teacher{
Some tech companies make a show of open-sourcing their models while declining to share the weights. That's a little like saying "we predict steering angle from curve and skew" but refusing to tell anyone the actual slope for either one.
}


== Multiple Regression in Pyret

=== Overview
Students use `regression-model-code` with multiple input columns to build a two-variable steering predictor, and `regression-model-S` to evaluate and compare models. They think critically about which variables to keep.

=== Launch
@slidebreak{Launch}

In the last lesson, we used `regression-model-code` with a list of one explanatory variable:

`regression-model-code(training-data, [list: "curve"], "steering-angle")`

Adding a second variable to the list is all it takes to go from a line to a plane!

`regression-model-code(training-data, [list: "curve", "skew"], "steering-angle")`

@slidebreak{Launch}

@slidebreak{Investigate}

@lesson-point{
Multiple regression is the same idea as linear regression, extended to any number of input variables. Each input gets its own slope (weight), and the model learns all of them at once to minimize error.
}

Multiple regression is the engine behind many real AI systems:

- ALVINN's hundreds of pixel weights
- Car insurance rates based on age, location, and driving history
- Weather forecasts based on temperature, humidity, pressure, and wind speed

The same three phases always apply when training and using a model: 

. Collect labeled data.
. Train a function.
. Apply the function to new inputs.


=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{ALVINN used hundreds of pixel values _and_ an obstacle distance as inputs. How does that connect to what we just did?}
@A{ALVINN's model had hundreds of parameters — one per pixel, each with its own learned weight — plus the obstacle distance. The same multiple regression principle applies, just at a much larger scale.}

@Q{What would it mean for a pixel to have a very _small_ weight in ALVINN's model?}
@A{That pixel barely affects the steering prediction — it probably captures a part of the image that rarely matters for navigating the road, like the sky.}

@Q{What would it mean for a pixel to have a very _large_ weight?}
@A{That pixel strongly influences the predicted steering angle — likely a part of the image that reliably signals road direction.}
}
