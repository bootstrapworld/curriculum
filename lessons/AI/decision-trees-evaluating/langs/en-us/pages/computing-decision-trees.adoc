== Building One-Level Decision Trees from Categorical and Quantitative Splits

*In the Definitions Area, define `tree-c1` and to be a 1-level tree using only the _categorical_ `"tail"` column, and click "Run":*

@hspace{1em}`tree-c1 = build-tree(training, [list: "tail"], "species", 1)`

@n Evaluate `tree-c1`. If an animals has no tail, it will predict: @fitb{5em}{@ifsoln{snail}}

@n Evaluate `dt-code(tree-c1)`. Where have we seen this code before? @fitb{}{@ifsoln{it's the code for `tail-classifier`!}}

@vspace{1ex}

*In the Definitions Area, define `tree-q1` to be a 1-level tree using only the _quantitative_ `"pounds"` column, and click "Run":*

@hspace{1em}`tree-q1 = build-tree(training, [list: "pounds"], "species", 1)`

@n Evaluate `tree-q1`. What split did it come up with? @fitb{}{@ifsoln{pounds < 9.55}}

@vspace{1ex}

*In the Definitions Area, define `tree-any` to be a 1-level tree using the _any_ column, and click "Run":*

== Building Multi-level Decision Trees from Categorical and Quantitative Splits

*Add the following definitions to your Definitions Area and click "Run":*

@hspace{1em}`tree-c2 = build-tree(training, [list: "tail"], "species", 2)` +
@hspace{1em}`tree-q2 = build-tree(training, [list: "pounds"], "species", 2)` +
@hspace{1em}`tree-any2 = build-tree(training, [list: "sex", "tail", "mammal", "swims", "pounds"], "species", 2)` 

@n In the code above, we allowed `tree-c2` to have a maximum depth of @fitb{2em}{@ifsoln{2}}. The resulting tree actually had @fitb{2em}{@ifsoln{1}} level. Why? +
@fitb{}{@ifsoln{once we split between animals with and without tails, there's no further sorting we can do with just the `"tail"` column.}}
