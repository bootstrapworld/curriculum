= Computing Decision Trees

@center{@show{(contract "build-tree" '("Table" "List" "String" "Number") "DecisionTree")}}

The `build-tree` function takes *four arguments* in its Domain:

[cols="1a,2a", frame="none", grid="none"]
|===
|
(a) The _training data_ on which to fit the tree +
(b) A list of all the column names to split on

|
(c) The name of the _target_ column, containing all the "answers" for the various categories +
(d) The _maximum depth_ the tree is allowed to be
|===

@link-instructions{Return to your copy of the @starter-file{decision-tree}.}
== Building One-Level Decision Trees

*Pyret can build one-level Decision Trees...*

@vspace{1ex}

*Based on a categorical column of our choosing.* 

In the Definitions Area, define `tree1` to be a 1-level tree using only the `"tail"` column:

@hspace{1em}`tree-cat1 = build-tree(training, [list: "tail"], "species", 1)`

@n Click "Run", and evaluate `tree-cat1`. What species will this classifier predict for animals *without* tails?  @fitb{}{@ifsoln{snail}}

@n Evaluate `dt-code(tree-cat1)`. Where have we seen this code before? @fitb{}{@ifsoln{it's the code for `tail-classifier`!}}

@vspace{1ex}

*Based on a quantitative column of our choosing.* 

In the Definitions Area, define `tree2` to be a 1-level tree using the `"pounds"` column:

@hspace{1em}`tree-quant1 = build-tree(training, [list: "pounds"], "species", 1)`

@n Click "Run", and evaluate `tree-quant1`. What split did it come up with? @fitb{}{@ifsoln{pounds < 9.55}}

@vspace{1ex}

*Based on whichever column Pyret determines will make the best predictions for our training dataset*.

@hspace{1em}`tree-any1 = build-tree(training, [list: "sex", "tail", "mammal", "swims", "pounds"], "species", 1)`

@n First, which column did you think would make the best @vocab{root node} for predicting `species` in the `training` dataset? @fitb{}{@ifsoln{`swims`}}

@n Click "Run", and evaluate `tree-any1`. Which column did Pyret choose? @fitb{}{@ifsoln{`swims`}}

== Building Multi-level Decision Trees
The trees we just built were restricted to a single level, but we can give Pyret any maximum number of levels to work with. 

@n Add the following definitions to your Definitions Area and click "Run". 

@hspace{1em}`tree-cat2 = build-tree(training, [list: "tail"], "species", 2)` +
@hspace{1em}`tree-quant2 = build-tree(training, [list: "pounds"], "species", 2)` +
@hspace{1em}`tree-any2 = build-tree(training, [list: "sex", "tail", "mammal", "swims", "pounds"], "species", 2)` 

@n What was the maximum number of levels defined for `tree-cat2`? @fitb{3em}{@ifsoln{2}} Evaluate `tree-cat2`. How many levels does it have? @fitb{3em}{@ifsoln{1}} Why? @fitb{}{@ifsoln{once we split between animals with and without tails, there's no further sorting we can do with just the `"tail"` column.}}

@n Add the following definitions to your Definitions Area and click "Run". 

@hspace{1em}`tree-quant3 = build-tree(training, [list: "pounds"], "species", 3)` +
@hspace{1em}`tree-quant4 = build-tree(training, [list: "pounds"], "species", 4)` +
@hspace{1em}`tree-quant5 = build-tree(training, [list: "pounds"], "species", 5)` +
@hspace{1em}`tree-any3 = build-tree(training, [list: "sex", "tail", "mammal", "swims", "pounds"], "species", 3)` +
@hspace{1em}`tree-any4 = build-tree(training, [list: "sex", "tail", "mammal", "swims", "pounds"], "species", 4)` +
@hspace{1em}`tree-any5 = build-tree(training, [list: "sex", "tail", "mammal", "swims", "pounds"], "species", 5)`
