== Supervised Learning

=== Overview
Having seeing the power -- and limits -- of unsupervised learning, students explore what it means to pair machine-learning algorithms with human-tagged data. They learn about centroids and the connection to measures of center, and use that concept to build recommendation systems and tag-based testing of images.


=== Launch
@slidebreak{Launch}

The similarity systems we've been discussing and building so far are an example of @vocab{unsupervised learning}. We're using a protractor in high dimensions to measure the angle between rays, and those rays are defined by numbers in a table. But that leaves out something really important: _human knowledge and experience_.

- All the sun images are different, but have *something* in common. If we want to find sun images, we want to find images similar to that *something*. But instead, we can only find images that are similar to a specific sun image. That means all the irrelevant differences in each image are included, leading to worse results.

- Google Images _also_ lets us type words. But we can only search-by-image, telling the computer to match a specific picture rather than asking it to find all the pictures with "grass".

@slidebreak{Launch}
These problems exist in the real world, too:

(1) Computers don't know that "Kung-Fu Panda" and "The Minecraft Movie" both star Jack Black
(2) Computers don't know that you like "Single Ladies", but not "Lemonade"
(3) Computers know that our snowy mountain pictures are similar, but has no idea that they both have snow in them.
(4) Computers can tell us that two jazz songs are similar, but can't just start a "jazz" station

We extracted the luminance, symmetry, and color information from the images by relying on mechanical analysis of the image data. But information about human preferences or groupings is something that requires human experience!

@slidebreak{Launch}
Of course, humans could _tell_ the computer this stuff by adding their knowledge and preferences to the dataset! This is where we shift from @vocab{unsupervised learning} to @vocab{supervised learning}.

You'll notice that our image table has extra columns for exactly this! `liked` and `disliked`, for example, are set to `false` - meaning no images have been rated one way or the other. `tags` are all the empty string, meaning nothing has been tagged yet.

Now a recommendation system is just:

- Calculate the centroids for liked and dislked documents
- Calculate the cosine similarities between every document and both centroids
- Recommendation strength is @math{\text{similarity-to-liked} - \test{similarity-to-disliked}}!

We've created a helper function that does exactly that, given a table and a list of columns:

@show{(contract "recommend" '(Table List<String>) "Table")}

The result is a table of all your documents, in descending order of recommendation-strength (strongest first).

Google's Image Search-by-word is just:

- Find all documents tagged with that word, and compute their centroid
- Compute the cosine-similarity of all the documents to that centroid
- Match accuracy is just @math{\text{similarity}}

We've created a helper function that does exactly that, given a table and a list of columns:

The result is a table of all your documents, in descending order of similarity to the centroid of the documents that match your tag.

=== Synthesize
@slidebreak{Synthesize}
You may recall the steps of @vocab{supervised learning} from a @lesson-link{regression-models-multiple, previous lesson}:

- Human-tagged data is recorded, demonstrating the knowledge to be learned
- The machine learning algorithm finds a *function* that generalizes (hopefully well!) from that data to new, unseen data
- The function is applied to new data.

ALVINN was given a table of inputs (images and depth maps), each row tagged with how a human turned the steering wheel. Then it found a function that would predict the steering wheel angle from new inputs, and used the function by applying it to new roads.

@QandA{
@Q{In our image search example, what is the human-tagged data for the computer?}
@A{The input table tagged with likes or dislikes, or words}

@Q{What algorithm is being used to find images?}
@A{Some kind of angle measure for similarity, followed by sorting}

@Q{Where is the generalization?}
@A{A new, un-tagged image added to database is still being compared to the pre-computed centroids}
}

@slidebreak{Synthesize}
@QandA{
@Q{When you tell Spotify that you like a song, what computations happen on Spotify's servers?}

@Q{When you tell Spotify that to make a Jazz station, what computations happen on Spotify's servers?}
@A{Google computes the centroid for every song *already tagged* with "jazz", then returns those alongside songs that are similar to the centroid}

@Q{When you search for "sun" on Google Images, what computations happen on Google's servers?}
@A{Google computes the centroid for every image *already tagged* with "sun", then returns those alongside images that are similar to the centroid}

@Q{How does Google know to filter out offensive images of extreme violence?}
@A{Google computes the centroid for every image *already tagged* as extremely violent, then filters our images that are similar to the centroid}
