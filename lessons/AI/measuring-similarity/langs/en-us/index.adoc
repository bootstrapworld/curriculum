== Categories, Attributes, and Spaces

=== Overview
Students explore the idea of _dimensionality_ -- that virtually any category of thing can be defined as having a set of attributes (e.g. - desserts can be defined by _chocolatiness_, _crunchiness_, and a long list of other things), and any member of that category can be graphed as a point in space (e.g. - plotting desserts on a _chocolatiness-v-crunchiness_ graph, or a graph based on any other pair of attributes).

Pyret has several functions for measuring similarity. They all work a little differently. Let's start with the simplest:

@hspace{1em}@show{(contract "simple-similarity" '("Table" "String" "List<String>") "Table")}

Pyret's `simple-similarity` function _compares every row in the table_ to the Row with the given ID, checking to see if the specified columns are identical. If they are, the similarity value is one. If not, the similarity value is zero. 

For example, if we wanted to know which restaurants had the same `price` as In-N-Out (whose ID is `I`) we would use the following code: +
@hspace{1em}@show{(code '(simple-similarity fast-food "I" (list "price")))}

== Other Forms of Similarity

=== Overview
Students explore other forms of similarity, using Euclidean distance and angle measures.

=== Launch
@slidebreak{Launch}
Checking for perfect equality is not a very useful way to look for alternatives to our favorite restaurant chain! 

Another option is to measure the _distance between points_.  In one dimension, this comparison is straight-forward: plot the values on a number line, and find the one that's closest to In-N-Out. Let's try it out.

@lesson-instruction{
- Complete the first section on @printable-exercise{distance-similarity.adoc}.
}

@slidebreak{Launch}

@QandA{
@Q{Did you find a clear answer as to which restaurant was most similar using 1 dimension? Why or why not?}
@A{No! The "most similar restaurant" to In-N-Out depended on the attribute:
  * McDonald's for price
  * Five Guys for burgers
  * Wendy's for fries
  }
@A{We have multiple, *independent* attributes for these restaurants!}
@A{Being similar for one attribute doesn't mean a restaurant is going to be similar for all the others.}

@Q{How could we measure the distance between two restaurants using more than one dimension?}
@A{We could graph the points using two dimensions and use the distance formula: @math{d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}}}
}

@teacher{
Your students will not need to do calculation with the distance formula in this lesson, because we are about to introduce the `distance-similarity` function.  Ideally, however, they would already have an understanding of how distance is computed in two dimensions.  If your students could use an introduction to or review of the distance formula and the Pythagorean Theorem, you might want to @lesson-link{distance, check out our lesson on distance in the coordinate plane.}
}

@slidebreak{Launch}

Pyret has a function for computing the `distance-similarity`, comparing every row in the table to a Row with a given ID. 

@hspace{1em}@show{(contract "distance-similarity" '("Table" "String" "List<String>") "Table")}

It will compute distance on a number line if the list we give it only includes a single column.

@hspace{1em} To compute a table with distances from In-N-Out (ID `"I"`) on the price scale: +
@hspace{1em} @show{(code '(distance-similarity fast-food "I" (list "price")))} 

But it can also compute distance in a plane described by any list of columns! 

@hspace{1em} To compute a table with distances from In-N-Out (ID `"I"`) on the Fries-Burgers plane: +
@hspace{1em} @show{(code '(distance-similarity fast-food "I" (list "burgers" "fries")))}

@slidebreak{Launch}

Our new `distance-similarity` formula is a huge improvement over `simple-similarity`! It's no longer an all-or-nothing situation. And, if In-N-Out isn't available, we have some suggested alternatives.

Let's take a look at how `distance-similarity` compares to human perception of similarity.

@QandA{
@right{@image{images/burger-fry-blank.png, 250}}
Some restaurants specialize in burgers, while others are all about the fries.
@Q{Roughly where on the graph would you expect to see restaurants that specialize in burgers plotted?}
@A{Top-Left}

@Q{Roughly where on the graph would you expect to see restaurants that specialize in fries plotted?}
@A{Bottom-Right}

@Q{Roughly where on the graph would you expect to see restaurants that are equally good (or bad!) at both burgers and fries?}
@A{Along the diagonal @math{y = x}}

@Q{What would you expect to be more similar: two places with terrible fries and good burgers, or two places with different strengths?}
@A{Most humans would generally say that the two places that specialize in burgers are more similar.}
}

@slidebreak{Launch}

@QandA{
@ifslide{@right{@image{images/burger-fry-blank.png, 250}}}
Suppose the `(fries, burgers)` scores for two burger joints are @math{(1, 7)} and @math{(1, 10)}. The distance between them is 3. +
Suppose the `(fries, burgers)` scores for two places with different strengths are @math{(2, 0)} and @math{(0, 2)}. The distance is @math{\sqrt(8) = ~2.8}

@Q{Which pair will `distance-similarity` say is closer?}
@A{The places with different strengths.}
}

There are a few problems with using the distance formula, however:

- It focuses on _count_, rather than _proportion_: restaurant scores that differ by 1 on the `fries` scale are _just as different_ as scores that differ by 2 on the `burgers` scale (both are one-fifth of the maximum points). The distance formula will treat the `burgers` difference as being "larger", because 2 > 1.
- It ignores _orientation_: As humans, we know that two places that focus on burgers are going to be a lot more similar that one that focuses on burgers and another that focuses on fries! But the distance formula doesn't care if two restaurants both specialize in the same thing or not -- it only cares about the distance _between_ the points, not _where those points are on the graph._

=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{What is an advantage of `angle-similarity` over `distance-similarity`?}
@A{Angle similarity focuses on the proportion of scores ("these restaurants' burgers are all _twice as good_ as the fries"), but distance focuses only on the scores themselves ("the distance between these two restaurants is 4.3"). One of these is much more useful to a person who wants a good burger!}
@A{Angle similarity takes _direction_ into account, and pays attention to points that are more on the burger side v. the fry side of the coordinate plane. Distance doesn't.}

This image represents a _projection_ of a 3D object into two, 2-dimensional planes. You can see that one shadow looks like a square and the other looks like a circle. Both are showing us a part of the story, and both are correct about the part they show. But the _real_ shape is 3-dimensional (3D).

*The same is true for our dataset!*

Our restaurants are described by 3 different attributes, so our "restaurant space" has three dimensions! Like the shadows in this image, our two scatter plots in "Price-Fry Space" and "Price-Burger Space" are just _projections_ of 3D points onto the Z-X and Z-Y planes.

@strategy{Isn't this Middle School Math?!}{

Yes, it is!

The same points and angles your students plot and compute in middle-school are sitting at the heart of the machine learning calculations that power so many things in the world today. Even the plagiarism detectors that might be checking their essays are computing angles. So if your students ask “When are we ever going to use this?”, you can tell them, “You already do, all the time.”

If you have older students who have learned some trigonometry, you might want to share that real machine learning systems use the cosine of the angle. There are two reasons for this:

- It’s simpler to compute the cosine directly. 
  * In fact, inside Pyret, `angle-similarity` actually computes the cosine _first_, then converts the result into an angle!)
- The angle itself is a somewhat awkward value to work with.
  * The idea that 0° means "most similar" and that 90° (the largest angle) actually means the _least_ similarity can be counter-intuitive.
  * In contrast, the cosine has a nice numeric range for angles between 0° and 90°, going from 1 to 0, which makes it convenient to use in various other mathematical settings. (Specifically, it's used in a process called @link{https://en.wikipedia.org/wiki/Gradient_descent, gradient descent} that is critical to modern AI. (Fun fact: gradient descent is actually just a real-world application of the @link{https://towardsdatascience.com/understanding-the-mathematics-behind-gradient-descent-dde5dc9be06e/, Chain Rule and Power Rule from Calculus}!)

For the purposes of this curriculum, you can safely ignore the difference between angle and cosine similarity! But, for students who are familiar with cosine and curious to explore, all of the starter files in our AI pathway have access to a `cosine-similarity` function.
}

=== Synthesize
@QandA{
@Q{It's obvious why `angle-similarity` works in two dimensions: you can use a protractor on a piece of paper if you have to! How could you explain why it still works in *three dimensions*?}
@A{All we need is two rays coming from the origin, forming a "V" shape like the bands in a slingshot. No matter how you turn and twist the "V", it will always define a plane on which we can use a protractor.}

@Q{Would the math still work in *four* dimensions? *Five*?}
@A{Let students discuss!}
@A{Just because we can't picture it, doesn't mean a computer can't calculate it!}
