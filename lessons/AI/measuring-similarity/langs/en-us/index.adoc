== Categories, Attributes, and Dimensions

=== Overview
Students explore the idea of _dimensionality_ -- that virtually any category of thing can be defined as having a set of attributes (e.g. - desserts can be defined by _chocolatiness_, _crunchiness_, and a long list of other things), and any member of that category can be graphed as a point in space (e.g. - plotting desserts on a _chocolatiness-v-crunchiness_ graph, or a graph based on any other pair of attributes).

@slidebreak{Investigate}

Pyret has several functions for measuring similarity. They all work a little differently. Let's start with the simplest:

@show{(contract "simple-similarity" '("Table" "String" "List<String>") "Table")}

Pyret's `simple-similarity` function _compares every row in the table_ to the Row with the given ID, checking to see if the specified columns are identical. If they are, the similarity value is one. If not, the similarity value is zero. 

For example, this code will produce a table with a new column, filled with 1s and 0s. Any row with a 1 is a restaurant that has the exact same `price` as the restaurant with ID `"6"` (In-N-Out).

@show{(code '(simple-similarity fast-food "6" (list "price")))}

@lesson-instruction{
- Try using `simple-similarity` with a variety of different columns
- Try using `simple-similarity` with a variety of different _restaurant IDs_

@Q{Why did `simple-similarity` give a 1 for McDonalds when we use the `price` column?}
@A{McDonalds `price` in our data table is the same as the `price` for In-N-Out.}

@Q{Why will we *always* get at least one perfect match in a `simple-similarity` table?}
@A{Because whatever ID we use will always be identical to itself!}

@Q{Is `simple-similarity` a _useful_ way to measure similarity?}

@Q{What would make a measure of similarity more useful?}
@A{We should at least be able to tell the difference between a restaurant that comes close (McDonald's) and a restaurant that's totally different (Outback Steakhouse)}

== Other Forms of Similarity

=== Overview
Students explore other forms of similarity, using Euclidean distance and angle measures.

=== Launch
@slidebreak{Launch}
Checking for perfect equality is not a very useful way to look for alternatives! Another option is to measure the _distance between points_.

In one dimension, comparing is easy: plot the values on a number line, and find the one that's closest to In-N-Out.

@lesson-instruction{
- Complete the first section on @printable-exercise{distance-similarity.adoc}.
}

@slidebreak{Launch}

@QandA{
@Q{When comparing restaurants in 1 dimension, is there one clear answer? Why or why not?}
@A{No! The "most similar restaurant" to In-N-Out depended on the attribute:
  * McDonald's for price
  * Five Guys for burgers
  * Wendy's for fries
}

@Q{How can this be?}
@A{We have multiple, *independent* attributes for these restaurants!}
@A{Being similar for one attribute doesn't mean a restaurant is going to be similar for all the others.}

@Q{How could we measure the distance between two restaurants using more than one dimension?}
@A{We could graph the points using two dimensions and use the distance formula: @math{d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}}}
}

@teacher{
We are about to introduce the `distance-similarity` function, so your students will not need to do calculation with the distance formula in this lesson. Ideally, however, they would have an understanding of how the program computes distance in two dimensions.  If they could use an introduction to or review of the distance formula and the Pythagorean Theorem, you might want to @lesson-link{distance, check out our lesson on distance in the coordinate plane.}
}

@slidebreak{Launch}

Pyret has a function for computing the `distance-similarity` of two points!

@show{(contract "distance-similarity" '("Table" "String" "List<String>") "Table")}

This function _compares every row in the table_ to the Row with the given ID, calculating the distance between the points specified by the columns. 

It takes in a list (that could include a single column or multiple columns) of data and produces a table with a new column, filled with the _distance_ between each restaurant and the restaurant with ID `"6"` (In-N-Out).! 

@lesson-instruction{
Test out:

- @show{(code '(distance-similarity fast-food "6" (list "price")))}
- @show{(code '(distance-similarity fast-food "6" (list "price" "fries")))}
- Try comparing `in-n-out` to _itself_ using all the columns
- Try some other comparisons!
- Then complete @printable-exercise{distance-similarity.adoc}.
}

@slidebreak{Launch}

Our new `distance-similarity` formula is a huge improvement over `simple-similarity`! Now it's no longer an all-or-nothing situation: if In-N-Out isn't available, we can still find alternatives.

@A{Bottom-Right}

@slidebreak{InvestigateR}

@QandA{
@ifslide{@right{@image{images/burger-fry-blank.png, 250}}}
Suppose the `(fries, burgers)` scores for two burger joints are @math{(1, 7)} and @math{(1, 10)}. The distance between them will be 3.
Suppose the `(fries, burgers)` scores for two places with different strengths are @math{(2, 0)} and @math{(0, 2)}. The distance between them will be @math{sqrt(8) = ~2.8}

@Q{Which pair will `distance-similarity` say is closer?}
@A{The places with different strengths.}
}

There are a few problems here. For example: Our distance algorithm doesn't take _scales_ into account.  `fries` and `price` are both scored 1-to-5 and `burgers` are scored 1-to-10. The distance between a score of 9 and 10 for burgers is larger than the one between a 4.0 and a 4.5 for fries, even though they are proportionally the same.

@right{@image{images/projection.webp, 300}}

This image represents a _projection_ of a 3D object into two, 2-dimensional planes. You can see that one shadow looks like a square and the other looks like a circle. Both are showing us a part of the story, and both are correct about the part they show. But the _real_ shape is 3-dimensional (3D).

*The same is true for our dataset!*

Our restaurants are described by 3 different attributes, so our "restaurant space" has three dimensions! Like the shadows in this image, our two scatter plots in "Price-Fry Space" and "Price-Burger Space" are just _projections_ of 3D points onto the Z-X and Z-Y planes.

@strategy{Isn't this Middle School Math?!}{

Yes, it is!

Did you know that geometry is at the heart of modern AI? This lesson shows how. The same points your students plot and angles they compute in middle-school are sitting at the heart of the machine learning calculations that power so many things in the world today. Even the plagiarism detectors that might be checking their essays on angles... are computing angles. So if your students ask “When are we ever going to use this?”, you can tell them, “You already do, all the time.”

The plot thickens, especially if you have older students who have learned some trigonometry. In practice, real machine learning systems don't _quite_ use angles. Instead, they use the cosine of the angle. There are two reasons for this:

- It’s simpler to compute the cosine directly. In fact, inside Pyret, `angle-difference` actually computes the cosine _first_, then converts the result into an angle!
- The angle itself is a somewhat awkward value to work with. The idea that 0° means "most similar" and that 90 (the largest angle) actually means the _least_ similarity can be counter-intuitive.
- In contrast, the cosine has a nice numeric range for angles between 0° and 90°, going from 1 to 0, which makes it convenient to use in various other mathematical settings. (Specifically, it's used in a process called @link{https://en.wikipedia.org/wiki/Gradient_descent, gradient descent} that is critical to modern AI. (Fun fact: gradient descent is actually just a real-world application of the @link{https://towardsdatascience.com/understanding-the-mathematics-behind-gradient-descent-dde5dc9be06e/, Chain Rule and Power Rule from Calculus}!)

For the purposes of this curriculum, you can ignore this difference. In particular, if your students have never even heard of the cosine, that's fine! For students who are familiar with cosine and curious to explore, all of the starter files in AI pathway have access to a `cosine-similarity` function.
}
