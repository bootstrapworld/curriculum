== Fitting Models for Age and Pounds

=== Overview
Students try fitting models to the animals dataset, comparing predictors for `age` and `pounds`.

=== Launch
@slidebreak{Launch}
In @lesson-link{fitting-models}, we you learned that a relationship between two variables will show up as a point cloud in a 2d scatter plot, and that this relationship can be described by a @vocab{model}.

@lesson-point{Most models are wrong. Some are _useful_.}

- A model for a 2d relationship is a @vocab{line of best fit}
- When it comes data from the real world, most models are @vocab{lossy}, meaning the line won't _perfectly_ fit the points. It will usually be at least a little "wrong".
- If the model is a good fit, however, the information lost is minimal and the predictions we make from the model can be useful.

Can we use `pounds` and `age` in our animals dataset to _predict_ how long it takes for an animal to be adopted?

Can we be more precise actually _predict_ how long it will take an animal to be adopted, based on these factors? And which one would give us a better prediction?

=== Investigate
@slidebreak{Investigate-DN}

As you already learned about @lesson-link{fitting-models}, our goal is to fit a line to each of these scatter plots, minimizing the error as much as possible.

@center{@big{@math{weeks(age)=(m_{age} \times age) + b_{age}}}}

- The function @math{weeks} has one variable named @math{age}, which explains a _portion_ of the adoption time (there might other factors besides age!)
- The portion explained by @math{age} is the slope @math{m_{age}} in our equation, with our y-intercept @math{b_{age}} representing the predicted value if @math{age = 0}

Suppose we're not interested in the graphs, and we _only_ want to know how well the model fits. How could we just get the @math{S}-value, without bothering with `fit-model`?

@show{(contract "simple-S" '("Table" "String" "String" "Number -> Number") "Number" "Consumes a table, the name of the input and target output, and a model of the form f(x) = ...")}


@QandA{
@Q{What is the *name* of this function?}
@A{`simple-S`}

@Q{How many things are in the *domain* of this function?}
@A{Four things: a table, two strings, and a function}

@Q{Why doesn't this function need to know the column for the labels of the points?}
@A{There *aren't any* points! We're just computing the @math{S}-value.}
}

This function is similar to `fit-model`! But instead of plotting points with labels, they just calculate our @math{S}-value and give it back to us.

=== Synthesize

@QandA{
@Q{Should we assume that either of our predictors will be generally accurate for another animal shelter?}
@A{No - this is only a handful of animals! Just not enough data to make a general claim. }
@A{No - there's no guarantee that another shelter even has the same species!}
@A{No - even with the same species, another shelter might be visited by humans with very different needs/interests, which could change the role that `age` and `weight` play in adoption time}

@Q{Suppose we did this same supervised learning on a dataset of millions of animals, from shelters all over the world. Could we assume those predictors are generally accurate?}
@A{It's definitely more likely, but there will still be variations from variables like differences between human preferences and differences between animals across shelters.}
}
== Simple Regression Models

=== Overview
Students are introduced to the concept (not algorithm) of @vocab{linear regression}, and learn how to interpret the @vocab{slope} and @vocab{y-intercept} of @vocab{predictor functions}.


=== Launch
@slidebreak{LaunchR}
@right{@image{images/lr-pounds-v-weeks.png, 300}}
Data scientists, statisticians, and machine learning engineers use a statistical method called @vocab{linear regression} to pinpoint linear relationships in a dataset. Linear regression is the math behind the @vocab{line of best fit}, which allows us to make predictions based on our data.

- In Machine Learning, the act of shrinking this data into a function is called @vocab{training}
- Linear regression finds straight-line patterns between a set of inputs and an _already-known_, target output

Think of each row in the animals dataset as a flash card, with `pounds` or `age` written on the front and `weeks` on the back. The computer is learning to predict the answer by identifying the patterns in our data. This form of learning -- where we already know the target values on -- is called @vocab{supervised learning}.

@teacher{
Sharp-eyed students might ask if there are other, _non-linear_ patterns that can be found via supervised learning. The answer is a resounding yes! In fact, the most important patterns in data when it comes to Machine Learning aren't linear at all. Algebra 2 is a great place to dig into non-linear functions, and @dist-link{courses/algebra-2/, we have a ton of material on that}.
}

@slidebreak{Investigate-DN}

@lesson-point{
In supervised learning, computers train on sample input-output pairs tagged by a human, to find the function that maps from input to output with the smallest amount of error.
}

Pyret has a function that does this learning for us, and its contract is the *same* as the contract for `scatter-plot`:

@show{(contract "lr-plot" '((table-name Table)  (label-col String) (x-col String) (y-col String)) "Image")}

Now we want Pyret to "learn" a function with the *best possible* values for @math{m_{age}} and @math{b_{age}}:

@center{@big{@math{weeks(age)=(m_{age} \times age) + b_{age}}}}

@ifslide{@right{@image{images/lr-pounds-v-weeks.png, 300}}}
@QandA{
@Q{What is the code for a row-consuming version of our `age` predictor?}
@A{`weeks(r) = (0.79 * r["age"]) + 2.3`}

@Q{What does the slope (the coefficient for `age`) tell us about what our predictor has _learned_?}
@A{An extra year of age predicts roughly 0.79 extra weeks at the shelter (about 5.5 days longer)}

@Q{What is the code for a row-consuming version of our `pounds` predictor?}
@A{`weeks(r) = (0.08 * r["pounds"]) + 2.5`}

@Q{What does the slope (the coefficient for `pounds`) tell us about what our predictor has _learned_?}
@A{An extra pound predicts roughly 0.08 extra weeks at the shelter (about half a day longer)}

@Q{Based on these slopes, which has the bigger _impact_ on adoption time: `pounds` or `age?`}
@A{`age`, because 0.79 is much larger than 0.08}

@Q{Which predictor _fits better_ (has a lower @math{S})?}
@A{`pounds`}

@Q{How could we possibly build an even _better_ predictor?}
@A{Split the dataset by species, and fit each one separately!}
@A{Somehow use `pounds` _and_ `age` together!}
}

=== Common Misconceptions
- *Don't forget to look at sample size!* A linear regression plot with an @math{r}-value of 0.999 is strong...but that's useless if it's a sample of just three data points!


=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{Why does it only make sense to use predictor functions for values that fall _within the range of the dataset_?}
@A{Because our function describes a pattern observed within that particular range. Lots of observable patterns do not continue outside of a certain range... for example, babies are supposed to triple in weight in their first year... but grownups definitely shouldn't!
}
@A{If students have completed @lesson-link{decision-trees-evaluating}, you can connect this back to trusting a classifier only on inputs that are similar to what it was trained on.}

@Q{Would it make sense to split the dataset by species, and do our supervised learning on each one separately?}
@A{Probably! Different species have wildly different sizes and lifespans. A 12 year old dog is quite old, while a 12 year old tarantula may have another 15 years to go.}
@A{In fact, lumping different subgroups together can hide a relationship, or even express the *wrong* relationship! This is known as Simpson's Paradox, which we talk about in @lesson-link{exponential4-simpsons-paradox}.}
}
