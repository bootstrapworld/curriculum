== Plotting Essays in Space
@objective{ai-bag-of-words}
@objective{ai-training-lossy}

=== Overview

Students review their progression from plotting numerical data to plotting image data, then consider the challenge of plotting human-written text. They apply the bag of words model from @lesson-link{measuring-similarity-2}, and confront the extra challenges that natural language presents.

=== Launch
@slidebreak{LaunchR}
Let's think back to our discussion of @lesson-link{measuring-similarity, similarity}. 

@lesson-point{
We *plotted documents in space* and used *geometry* to calculate something approximating "similarity".
}

@right{@image{images/3-d-comparison-w-lines.png, 300}}

- A table with @math{n} numeric attributes (e.g., number of green-dominant pixels) can be graphed as points in @math{n}-dimensional space
- The location of these points in space can be used to compute different measures of similarity (equality, distance similarity, angle similarity)
- Machine Learning Engineers typically use angle measure, because it more closely reflects human intuition for things clustered on an axis being more similar to each other than things clustered on different axes.

@slidebreak{Launch}
If the input data _isn't_ numeric, we can compute numbers anyway!

- In our image search examples, we generated numeric attributes like symmetry, luminance, and entropy
- We also generated very long Strings of `COLOR-NAMES`, listing each of the dominant primary colors in pixel order.
- We turned our ordered string of colors into an _un-ordered_ count of the number of red-, green-, and blue-dominant pixels using a @vocab{bag of words model}
- Each count became another attribute, allowing us to plot images in "color-space"

@QandA{
@Q{How is a bag-of-words model "lossy"? What details of the original training data does it lose?}
@A{The order of the words}
}

@slidebreak{Launch}

Can we compute the similarity of _anything_ that way? Think about:

- Drivers' License Photos
- Essays
- Recipes

== Data Normalization
@objective{ai-data-normalization}
=== Overview
Students explore the importance of @vocab{data normalization}, when we organize data to follow a standard pattern.

=== Launch
@slidebreak{LaunchR}
@right{@image{images/bag-of-colors.png, 250}}
When working with images, we used a tool that looked at the _frequency of colors_ in the image:

@show{(contract "add-bag-cols" '("Table" "String") "Table")}

The `add-bag-cols` function dropped all the information about the _order_ of colors, and instead just counted how many of each pixel appears in the image.

*How could we use this with our essays?*

@slidebreak{Launch-DN}

Now that we have dozens of columns, we need to some help choosing which columns to use for calculating similarity!

@show{(contract "all-cols-similarity" '("Table" "String") "Table")}

This function consumes a Table and the ID of a target Row, and then _uses all the numeric columns_ to compute the angle-similarity. For example: `all-cols-similarity(bags, "K")` will find the angle-similarity of every document to "K", using every column containing numbers.

=== Investigate

@slidebreak{Investigate}

@QandA{
@Q{What does a highest-possible score of 1 tell us, if we're using @vocab{bag of words models}? }
@A{Two essays use the exact same words, the exact same number of times}

@Q{What does the lowest-possible score of 0 tell us, if we're using @vocab{bag of words models}? }
@A{Two essays that don't using any of the same words at all}

@Q{Which essay was most similar?}
@A{The elephant essay}

@Q{_How_ similar was it?}
@A{Roughly 0.8653}

@Q{What about the second-most-similar essay. What was it about, and how similar was it?}
@A{The Polar Bear essay was roughly 0.4516}
}

@slidebreak{Investigate}
We definitely found the most similar essay, but it's a little strange that the Polar Bear essay was computed to be similar at all! And it's not just slightly similar, either.

*Why do you think we're seeing similarity about such different essays?*


@slidebreak{Launch}
The problem is that our data is "messy":

- Punctuation: `areas` and `areas.` aren't different words
- Capitalization: `By` and `by` aren't different words

This means our counts for these word are off, and that we're creating extra, meaningless dimensions!

@slidebreak{Launch}
But there's another problem here, too!

- Common Words: `the`, `is`, and `a` are "similar" across all the essays

These are called @vocab{stopwords}, and they show up in almost any sentence.  These words create many dimensions in which the essays are close together, raising the similarity score between two essays simply because they're both English! In our example, the elephant essay was able to rise above the noise and was obviously the most similar, but in other situations we might get a lot of false positives!


@slidebreak{Investigate-DN}
Addressing this "messiness" is called @vocab{data normalization}. Machine Learning Engineers and Data Scientists care a lot about normalizing their data in order to cut out the irrelevant noise and focus on what matters. We've provided three normalization functions for you to experiment with:

@show{(contract "lowercase" '("String") "String")}

@show{(contract "remove-punct" '("String") "String")}

@show{(contract "remove-stops" '("String") "String")}

=== Synthesize
@QandA{
@Q{Did normalizing our data give us better results? Why do they seem "better"?}
@A{Yes! The elephant essay is still really similar, but the others are much *less* similar}

@Q{Explain why normalizing the data was important here}
@A{Removing garbage data and common words allowed us to focus our comparison on only the words that matter, and count them properly.}

== Detecting Plagiarism
@objective{ai-plagiarism-detector-ethics}
=== Overview
Armed with their similarity detection, students explore the how that algorithm can be used to detect plagiarism. Now that they know how it works, they consider limitations of the algorithm and when it might not be ethical for teachers to use it.

=== Launch
@slidebreak{Launch}
A @vocab{training corpus} is a collection of data used to train AI/ML @vocab{models}, enabling them to learn patterns and make predictions.

For a plagiarism checker, the corpus is the set of documents previously fed into the checker.

@QandA{
@Q{What sorts of documents make up the training corpus of an _effective_ plagiarism detector? List as many as you can.}
@A{The corpus would likely include:
  * essays written and submitted by students currently in the class
  * essays written and submitted by students previously in the class
  * Wikipedia articles
  * articles on relevant topics that are available on the internet, etc.}
}

Suppose a teacher builds a plagiarism detector for short animal essays. She turns paragraphs for ten different animals into bags of words and adds them to the model. This setup is done only _once_, and can be reused for any student submission.

The @vocab{training} phase is where the enormous corpus of essays, wikipedia articles, etc is converted to a bag-of-words for each document, and stored. 

*Our `bags` table is our model!*

=== Investigate
@slidebreak{Investigate}

@lesson-point{Once a @vocab{model} is @vocab{trained}, the corpus can be queried as many times as we want without having to repeat any of the work done during training!}

@slidebreak{Investigate}
To catch plagiarism across _any_ essay, a teacher needs a model trained on a _massive_ document collection. Processing this large corpus creates a complex, high-dimensional space where each unique word adds a new dimension.

@QandA{
@Q{If a teacher feeds twenty 500-word student essays into a plagiarism detector to use as a training corpus, about how many dimensions will the resulting model have?}

@A{Students should provide a wide range of estimates.}
@A{The largest possible estimate would be 10,000 dimensions (20 essays multiplied by 500 words) --but it is not a good estimate, because we commonly repeat and reuse words like "the", "and", "a", and so on.}

@A{Before making an estimate, students might have clarifying questions, like:
  * Did all of the students write about the same topic?
  * How sophisticated is the student writing?
  * Did all students actually write 500 words?
}

@A{A reasonable prediction would probably be that there would be at least a few hundred dimensions in the model.}
}

=== Synthesize
@slidebreak{Synthesize}

@QandA{
@Q{What strategies did you use to complete the matching activity? Were any of the scenarios especially challenging to match?}

- In Class A, 3rd graders are asked to write 1-paragraph essays about zebras
- In Class B, high school freshmen are asked to write a 3-page paper about their favorite TV show

@Q{Is it *equally ethical* for the teachers in both classes to use plagiarism detectors with their students' work? Why or why not?}
@A{Definitely not!}
@A{3rd graders have smaller vocabularies, if all the essays are about zebras the number of unique words will be small (they'll all use "Zebra", "Stripes", and "Africa", for example). And being just one paragraph long, there's not a ton of room for the frequency of words to differ! It's very likely there will be false positives in Class A, and a plagiarism detector is just not appropriate.}
}
